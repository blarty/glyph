@Exportable

The @Exportable annotation is used to generate three things from an object you wish to export as a remote object:
  • A smart-proxy annotation that supports Constraints
  • A Proxy Creator that can choose between a normal ans constrained version of a proxy
  • A default exporter configuration file for your remote object

Smart proxies

A smart proxy is serializable class that contains a reference to an exported remote class. By encapsulating the remote reference, the smart proxy can execute part or all of a particular method on the client, and can be used for things like caching or references that need not be retrieved from the service each time they are called. The @Exportable annotation can automatically create a smart proxy that implements RemoteMethodControl for secure method calls, and will also write a Creator class for the proxy, and an ExporterManager definition.
By making a single call to an ExporterManager, an object can be exported, wrapped in a smart proxy, and managed all at once, with the calling class only needing to keep a reference to the Uuid that it passed to the method call.
