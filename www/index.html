Glyph is a set of utilities and annotations to speed up development for Jini-enabled applications.

J2SE5.0 annotations are used for automatic service creation and creation of smart proxies for remote objects, and generation of associated Jini configuration files.

A number of utilities have been written to help manage exported objects, and ease service initialization.

In development, there are other helpful classes such as service filters, and Pack200 URL handlers that can significantly reduce the size of a downloaded jar file when working with mobile code.

The glyphs

  • @Service
  • @Client - Released soon in 0.3
  • @Exportable
  • @ServiceUI
  • @Injection
  • @LocalService
  • @LeasedResource - Released soon in 0.3

Generation

Glyph can create a number of source files, configurations, build scripts and wire-ups including

  • Service Binding implementation
  • Client Binding implementation
  • Service and Service Starter configuration files
  • Support for Activatable, Non-activatable and main() service startup
  • Shell scripts
  • ServiceUI Factories
  • Local Services
  • Administration Proxies
  • Default attributes
  • Dependency Injection wiring
  • Attribute Builders
  • Full leasing system including Landlord and Reaping for individual LeasedResources
  • classdepandjar ant scripts for automatic creation of dl files

Glyph takes care of all of these things automatically......although there is much more to be done

Utilities

Glyph has a number of utilities and helper classes including:
  • ExporterManager
  • HyperJarBuilder

Infrastructure

Glyph also adds a number of under-the-cover utilities which can be applied to many jini services without code or configuration changes. For the moment there are two URL handlers (hjar and pack) which enable the use of internal jar formats and Pack200 compressed jars for use in mobile code These are patched into jsk-resources
