Patching the Glyph URLs

The Two URL handlers that come with Glyph can be patched into the jsk-resources jar, and should be patched into every jsk-resources jar on your federation The patch Installer can be downloaded from here There are two forms of the patch called the sun and the glyph patches respectively.
Both install exactly the same function but in different packages:

The sun patch installs the hjar and pack handlers in the sun.net.www.protocol namespace and is therefre automatically detected and available for use in any jini application running on a Sun VM, the glyph patch installer the handlers in the org.jini.glyph.url namespace and requires you to add a system property of :
-Djava.protocol.handler.pkgs=org.jini.glyph.url

To install the patch use the following, and ensure that jar.exe is on your path:
java -jar glyphpatch.jar <jini-install-directory> <patch-file>
where patch file is either 'sun.net.www' or 'org.jini.glyph'
i.e. java -jar glyphpatch.jar /usr/local/java/jini2_1 sun.net.www

Note On windows if you have a directory with a space in it, please use the DOS 8.3 filename i.e. for Incax Eclipse plugin use java -jar glyphpatch.jar c:\docume~1\<username>\.incax\eclipse-ce sun.net.www

The old version of your jsk-resources will be called jsk-resources.jar.back
